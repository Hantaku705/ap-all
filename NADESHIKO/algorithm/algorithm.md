# TikTok アルゴリズム解説

TikTokのFor You Page（FYP）アルゴリズムの仕組みと攻略法。

---

## アルゴリズムの基本構造

### For You Page（FYP）の仕組み

```
投稿 → 初期配信（100-500人）→ 評価 → 拡散 or 停止
         ↓
    エンゲージメント測定
    ・視聴完了率
    ・いいね率
    ・コメント率
    ・シェア率
    ・保存率
```

### 配信の段階

| 段階 | 配信数 | 条件 |
|------|--------|------|
| 第1段階 | 100〜500人 | 投稿直後（フォロワー + 興味関心マッチ） |
| 第2段階 | 1,000〜5,000人 | 第1段階で高エンゲージメント |
| 第3段階 | 10,000〜100,000人 | 第2段階で継続的に高評価 |
| バイラル | 100,000人以上 | 複数指標で上位パフォーマンス |

---

## 重要指標（ランキングシグナル）

### 優先度順

| 優先度 | 指標 | 説明 | 目標値 |
|--------|------|------|--------|
| 1 | **視聴完了率** | 動画を最後まで見た割合 | 70%以上 |
| 2 | **リプレイ率** | 繰り返し視聴された割合 | 高いほど良い |
| 3 | **シェア率** | 他者に共有された割合 | 1%以上 |
| 4 | **コメント率** | コメントがついた割合 | 0.5%以上 |
| 5 | **いいね率** | いいねされた割合 | 5%以上 |
| 6 | **保存率** | 保存された割合 | 1%以上 |
| 7 | **フォロー率** | 動画からフォローされた割合 | 0.5%以上 |

### 視聴完了率の重要性

> **視聴完了率が最も重要**
>
> TikTokは「ユーザーを長くアプリに留める」ことが目的。
> → 最後まで見られる動画 = ユーザーを留める動画 = 優遇

## 移動平均の法則

### 概念

> 結局次の動画は前の動画のパフォーマンスを引きずるので、移動平均のようなグラフイメージ

```
過去7-14本の平均再生数
    ↓
アカウントの「期待値」としてアルゴリズムが認識
    ↓
次の動画の初期配信数に影響
```

### 影響

| 状態 | 移動平均 | 次の動画への影響 |
|------|----------|-----------------|
| 好調 | 高い | 初期配信数が多い → バズりやすい |
| 低迷 | 低い | 初期配信数が少ない → 回復困難 |

### 対策

| 状態 | 対策 |
|------|------|
| 低迷時 | **企画動画を投下して突破**（移動平均リセット） |
| 好調時 | 投稿頻度を維持して勢いを保つ |

## アカウント質スコア

### なぜアカウント質が重要か

> **投稿アカウントのおすすめ欄がターゲットと同じなら理想**
>
> TikTokは「このアカウントはどんなジャンルのアカウントか」を視聴行動から判断している。
> → 美容アカウントなら、おすすめ欄が美容動画で埋まっている状態が理想。

### 構成要素

| 要素 | 説明 | 改善方法 |
|------|------|----------|
| **アカウント年齢** | 古いほど信頼性UP | 時間 |
| **投稿頻度** | 継続的な投稿 | 毎日投稿 |
| **エンゲージメント履歴** | 過去の平均パフォーマンス | 高品質動画 |
| **視聴行動** | アカウント自体の視聴パターン | 1日30分視聴 |
| **コミュニティ参加** | コメント・いいね活動 | 積極的に交流 |
| **おすすめ欄の状態** | ターゲットジャンルの動画で埋まっているか | ジャンル動画を見続ける |

### 新規アカウントの育成（最初の2週間）

> **最初の2週間はターゲットの動画を見まくる**

| 期間 | やること | 目的 |
|------|----------|------|
| 1-2週目 | ターゲットジャンルの動画を視聴し続ける | おすすめ欄をターゲット層と同じ状態にする |
| 3週目〜 | 投稿開始 | ジャンル認定されたアカウントからの投稿 |

**例**: 美容アカウントなら美容動画を見まくり、おすすめ欄が美容動画で埋まるようにする。

### アカウントがダメな場合

> **アカウントが微妙なら作り直した方がいい**

| 兆候 | 対応 |
|------|------|
| 再生数が継続的に低い | 新規アカウント作成を検討 |
| おすすめ欄がターゲットと違う | 新規アカウント作成 |
| シャドウバンの疑い | 新規アカウント作成 |

### スマホ農場（Phone Farm）によるアカウント質向上

複数端末でアカウントの視聴行動を最適化し、アルゴリズム評価を向上させる手法。

**中国での運用**: 自動スクロール等でターゲット動画を大量視聴し、おすすめ欄を最適化。

## デバイス・ネットワークの影響

### 1スマホ1アカウントの原則

> **同じスマホで複数アカウント登録はやめるべき**

| 状況 | リスク |
|------|--------|
| スマホにA・B両アカウント登録 | Aで見た動画がBのおすすめに連動 |
| 結果 | アカウントのジャンル特性が混ざる |

**推奨**: 1アカウント = 1スマホ

### WiFiの影響（バン連鎖）

> **同じWiFi下でバンが発生すると、他のアカウントも影響を受ける**

| 状況 | リスク |
|------|--------|
| 同一WiFiでアカウントAがバン | 同一WiFiのアカウントB・Cもバン対象になりやすい |

**対策**:
| 対策 | 説明 |
|------|------|
| WiFi分離 | アカウントごとに異なるネットワークを使用 |
| モバイル回線 | 重要アカウントはモバイル回線を使用 |
| VPN | 必要に応じてIPを分散（※TikTokのVPN検知に注意） |

## コンテンツ最適化

### 動画構成の黄金律

```
0-1秒: フック（Hook）     → 離脱防止
1-3秒: 価値提示（Value）  → 期待形成
3秒-: 本編（Content）     → 視聴維持
最後: CTA（Call to Action）→ エンゲージメント誘導
```

### フックのパターン

| パターン | 例 |
|----------|-----|
| 質問型 | 「〇〇って知ってる？」 |
| 驚き型 | 「これ見て！」「やばい」 |
| 問題提起型 | 「〇〇で悩んでない？」 |
| 結果先出し型 | ビフォーアフターを最初に見せる |
| 対立型 | 「〇〇 vs △△」 |

### 視聴完了率を上げるテクニック

| テクニック | 説明 |
|-----------|------|
| **ループ動画** | 終わりと始まりを繋げて繰り返し視聴させる |
| **テンポ感** | 間延びしない編集（ジャンプカット） |
| **テキスト** | 音なしでも理解できる字幕 |
| **期待感** | 「最後に衝撃」など煽り |
| **短尺** | 15-30秒が最適（完了率上げやすい） |

## シャドウバンと回避

### シャドウバンの兆候

| 兆候 | 説明 |
|------|------|
| 再生数が急激に減少 | 通常の1/10以下 |
| FYPに表示されない | フォロワーにしか届かない |
| ハッシュタグ検索で出ない | 検索結果から除外 |

### 原因

| 原因 | 説明 |
|------|------|
| コミュニティガイドライン違反 | 過激なコンテンツ |
| スパム行為 | 大量フォロー/いいね |
| 著作権違反 | 音楽・映像の無断使用 |
| 同一コンテンツ重複 | 同じ動画を複数投稿 |

### 回避策

| 対策 | 説明 |
|------|------|
| オリジナル音源使用 | 著作権フリー音源 or TikTok提供音源 |
| 動画の差別化 | 反転、カット、順番入れ替え（アルゴリズム回避） |
| 自然なエンゲージメント | 急激なフォロー増を避ける |
| ガイドライン遵守 | 過激表現を避ける |

### 5^6理論（素材組み合わせ戦略）

> **同じ動画や素材を使ったらおすすめに乗りにくくなる**
>
> → 同じ素材でも複数組み合わせるとバン率が下がる

**理論**:
```
30秒動画 = 6シーン（5秒×6）
各シーンに5種類の素材を用意
→ 組み合わせパターン = 5^6 = 15,625通り
```

| 方式 | バン率 | 通過率 |
|------|--------|--------|
| 同一素材そのまま | 高い | 低い |
| 5^6組み合わせ | 低い | **約60%** |

**実践方法**:
1. 30秒動画を5秒ごと6シーンに分割
2. 各シーンで5パターンの素材を撮影/作成
3. ランダムに組み合わせて複数バージョン生成

### アーカイブの無意味さ

> **投稿したらデータとして保存されるので、アーカイブしても意味がない**

| 状態 | TikTokのデータ |
|------|---------------|
| 公開中 | 保存済み |
| アーカイブ後 | **保存済み（変わらない）** |
| 削除後 | 保存済み（認識モデルの学習データとして残る可能性） |

**結論**: 一度投稿した動画のデータはTikTok側に残るため、アーカイブは「見えなくする」だけで、アルゴリズム上の履歴はリセットされない。

## TikTok Shop（TTS）とアルゴリズム

### GMV MAXの仕組み

```
商品タグ付き動画
    ↓
購入発生 → アルゴリズムが「売れる動画」と認識
    ↓
さらに配信拡大（好循環）
```

### 初動ハック

| ステップ | 内容 |
|----------|------|
| 1 | 身内に20件購入依頼 |
| 2 | コメントで商品FAQ化（スクリプト用意） |
| 3 | 毎日コメント返信でエンゲージメント維持 |

**効果**: 初動で「売れる動画」認定 → GMV MAXで拡散

### 低品質動画の排除（TTS独自）

> **顔出ししてないからヒットしない、というわけではない**
>
> TikTok Shopでは低品質動画の排除を行っており、複雑な認識モデルが組まれている。

| 排除対象 | 説明 |
|----------|------|
| 顔出しなし | 顔が映っていない動画 |
| 静止画 | 動きのない画像ベース動画 |
| AI読み上げ | 機械音声によるナレーション |
| 非ネイティブ動画 | 明らかに機械生成・低品質なコンテンツ |

**重要**: 単純に「顔出ししていないからNG」ではなく、複合的な品質判定が行われている。

## 検索アルゴリズム（SEO）

### TikTok検索の重要性

> TikTokは「検索エンジン」としても使われている
> （特にGen Z）

### 検索順位の決定要因

| 要因 | 説明 |
|------|------|
| キャプション | 検索キーワードを含める |
| 音声認識 | 動画内で話した言葉も認識 |
| テキスト | 動画内テロップも認識 |
| エンゲージメント | 検索結果からのクリック率・視聴完了率 |

### 対策

| 対策 | 内容 |
|------|------|
| **検索上位コンテンツを真似る** | 構成・キーワードを参考に |
| キーワード入れ込み | キャプション + 音声 + テロップ |
| 回答型コンテンツ | 「〇〇とは」「〇〇のやり方」 |

## 分析と改善

### TikTok Analyticsで見るべき指標

| 指標 | 説明 | 目標 |
|------|------|------|
| 視聴完了率 | 最重要指標 | 70%以上 |
| 平均視聴時間 | 動画の何秒まで見られたか | 動画の80%以上 |
| トラフィックソース | FYP / フォロー / 検索 / その他 | FYP比率50%以上 |
| フォロワー増減 | 投稿ごとの増減 | 純増維持 |

### PDCAサイクル

```
1. 投稿
    ↓
2. 24-48時間後にAnalytics確認
    ↓
3. 視聴完了率・トラフィックソース分析
    ↓
4. 次の投稿で改善
```
