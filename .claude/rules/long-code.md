# 長文コードの取り扱い

## ルール

長文のコード（10行以上）をユーザーに提供する場合:

1. **ターミナル上での表示を避ける** - コピー時に余白が入りエラーの原因になる
2. **ファイルに書き出す** - 指定されたパス、または適切な場所に保存
3. **ファイルパスを伝える** - ユーザーがそこからコピーできるようにする

## 例

### NG: ターミナルに長文コードを表示

```
以下のコードをコピーしてください:

function example() {
  // 長いコード...
}
```

### OK: ファイルに保存

```
コードを以下に保存しました:
/path/to/file.js

このファイルの内容をコピーしてください。
```

## 対象

- JavaScript / TypeScript
- Python
- Google Apps Script
- SQL
- その他すべてのコード

## 理由

- ターミナルからのコピーは余白やフォーマットが崩れやすい
- IDEやエディタで開けば正確にコピーできる
- バージョン管理もしやすい
