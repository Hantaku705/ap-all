# brief/ - クライアント共有用ドキュメント

プロジェクトの目的・課題・仮説を整理したクライアント共有用ドキュメント。

---

## サブフォルダ一覧

| フォルダ | 説明 |
|---------|------|
| `brand/` | ブランドマスタ（10ブランド） |
| `init/` | プロジェクトブリーフ |
| `issue/` | 核心的な問い（Issue 1/2） |
| `hypothesis/` | 仮説と検証結果 |

---

## 作成順序

```
1. brand/brand.md      ← 公式サイト画像からブランド名抽出
2. init/init.md        ← ユーザーと目的・課題を擦り合わせ
3. issue/issue.md      ← init.md から核心的な問いを整理
4. hypothesis/hypothesis.md ← brand.md × issue.md から仮説を立案
```

---

## 各フォルダの詳細

### brand/（ブランドマスタ）

- 10ブランドの一覧
- UGC/検索での呼ばれ方
- 検索キーワード設計

### init/（プロジェクトブリーフ）

- **目的**: 商品単位 → 生活文脈単位への転換
- **課題**: サイロ化、生活者視点欠如、根拠不足
- **Q1-Q4**: 明らかにしたいこと

### issue/（核心的な問い）

- **Issue 1**: ブランド間の双方向性（正の相関・負の相関・カニバリ）
- **Issue 2**: 全体最適のドライバー（キーブランド・訴求軸・投資配分）

### hypothesis/（仮説マトリクス）

- ブランドペアの相関予測
- H1: 正の相関予想ペア
- H2: カニバリ予想ペア
- 検証結果
